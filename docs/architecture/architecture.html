// request lifecycle

1
HTTP Request arrives
Client sends request with Authorization: Bearer <token> header
2
ValidationPipe runs
Body is validated against DTO — invalid fields are rejected (whitelist + forbidNonWhitelisted)
3
JwtAuthGuard verifies token
JWT is decoded and verified with JWT_SECRET — user is attached to request
4
RolesGuard checks permissions
Reads @Roles() metadata from endpoint — compares with user.role from token
5
Controller delegates to Service
Handler receives clean, typed data — business logic runs in the Service layer
6
Prisma executes query → response
Type-safe query runs against PostgreSQL — result is serialized and returned
02   Use Cases

Actor Capabilities

Guest Unauthenticated
  • Register a new account
  • Login with email + password
  • Receive JWT access token
  • Browse Swagger documentation
User Authenticated
  • Create new tasks
  • List own tasks
  • View a specific task by ID
  • Update task title, description, status, priority
  • Soft-delete own tasks
Admin Full Access
  • All USER capabilities
  • View all users in the system
  • View tasks from any user
  • Delete any task or user
  • Assign or change user roles

// api endpoints

Method Endpoint Description Access
POST /auth/register Register new user Public
POST /auth/login Login, returns JWT Public
GET /tasks Get own tasks 🔒 USER
GET /tasks/:id Get task by ID 🔒 USER
POST /tasks Create a new task 🔒 USER
PATCH /tasks/:id Update task fields 🔒 USER
DELETE /tasks/:id Soft-delete task 🔒 USER
GET /users List all users 🛡 ADMIN
DELETE /users/:id Delete a user 🛡 ADMIN
